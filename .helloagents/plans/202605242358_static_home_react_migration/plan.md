# aiapi114 首页按静态 HTML 改造 — 实施规划

## 目标与范围
本次改造把静态 HTML 首页沉淀成正式 React 首页，而不是把静态页简单嵌入 iframe。正式首页 `/` 需要复刻静态页的内容、视觉、主题切换和滚动动效，并接入系统公告、多语言、模型状态等项目已有能力。静态 HTML 首页作为 SEO 落地页继续保留在 `/prototypes/home-html/`，作为视觉参考和后续 SEO 资产。

## 架构与实现策略
推荐方案：以 `src/features/home` 为边界重建首页组件，并将静态页内容拆成数据、组件、样式 token 和业务适配层。

关键策略：
- `Home` 入口保留现有自定义首页内容逻辑：如果后台配置了自定义 `home_page_content`，仍优先渲染自定义内容；无自定义内容时渲染新版默认首页。
- 新版首页内部采用 feature-based 结构：页面容器、区块组件、业务 hooks、静态内容配置、主题动效与工具函数分离。
- 文案进入 i18n，结构化内容进入 `content` 或 `constants` 文件，避免组件内长文本硬编码。
- 公告、模型状态通过 hook 接入 API，失败时使用静态页中的安全兜底，不阻塞首页主体。
- 样式优先使用 Tailwind 与 CSS 变量；Hero 动态插图、全屏主题切换动画、滚动入场效果使用少量 scoped CSS，不引入新框架。
- 静态页中的 SVG 图标和 Hero 插图逻辑优先重写为 React SVG / CSS 组件，避免运行时依赖静态 HTML sprite。

## 领域语言
- “静态 HTML 首页”：`/prototypes/home-html/` 下的 SEO 落地页和视觉参考，不等同于正式 React 首页。
- “正式首页”：TanStack Router 的 `/` 页面，由 `src/features/home` 渲染。
- “公告”：系统公告数据；首页顶部展示最新有效公告或兜底公告。
- “模型状态摘要”：首页中的轻量状态预览，不是 `/status` 完整状态页。
- “主题切换动画”：黑夜 / 白天切换时的全屏视觉过渡，需接入全局主题状态。

## 完成定义
功能完成时必须同时满足：
- `/` 首页展示静态 HTML 首页的全部核心内容与交互，不低于静态页的信息完整度。
- `/prototypes/home-html/` 仍可访问，文件未被删除，静态 SEO 页样式不被破坏。
- 系统公告在首页顶部显示真实数据或明确兜底，点击入口行为清晰。
- 主题切换能改变正式首页全局视觉，并出现可识别的全屏过渡动画。
- 多语言切换后首页主要内容随 i18n 更新。
- 模型状态摘要消费 `/api/uptime/status` 或现有领域转换逻辑，并提供异常/空数据状态。
- 桌面 1440px、移动 390px 至少各完成一次截图验收；黑夜与白天主题都要检查。
- `typecheck`、首页相关测试、模型状态相关测试、构建或等效项目验证通过。

验证主路径：`test-first`。先补齐可测试的数据映射、状态归一化和关键交互测试，再实现 UI 与业务接入。

Reviewer 关注：
- 静态 SEO 页是否被保留。
- 组件边界是否清晰，是否出现超大组件或静态 CSS 直接搬运。
- 主题、i18n、公告、模型状态是否沿用现有项目能力。
- 错误、空数据、加载和减弱动效状态是否覆盖。

Tester 关注：
- `/` 首页在黑夜 / 白天、中文 / 英文、桌面 / 移动下可用。
- 公告接口成功、空数据、失败三类状态可用。
- 模型状态接口成功、空数据、失败三类状态可用。
- 主题切换动画不会导致页面卡死、遮挡不可恢复或焦点丢失。

## 文件结构
预计涉及：
- `web/default/src/routes/index.tsx`：保持首页路由入口，必要时只做轻量接线。
- `web/default/src/features/home/index.tsx`：保留自定义首页逻辑，接入新版默认首页。
- `web/default/src/features/home/components/...`：重建首页区块组件。
- `web/default/src/features/home/hooks/...`：公告、模型状态摘要、主题切换动效、滚动入场。
- `web/default/src/features/home/lib/...`：模型状态摘要映射、公告归一化、复制链接等纯逻辑。
- `web/default/src/features/home/content/...` 或 `constants.ts`：静态页内容结构化配置。
- `web/default/src/features/home/styles...` 或全局样式入口：scoped 首页主题 token、Hero 插图、全屏主题切换动画。
- `web/default/src/i18n/locales/*.json`：新增首页多语言文案。
- `web/default/src/features/model-status/lib/status-view.ts`：如现有映射无法复用，补充导出轻量摘要工具。
- `web/default/public/prototypes/home-html/*`：只作为保留与对照，不进行破坏性修改。

## UI / 设计约束
本次 UI 决策优先级：本方案 `plan.md` → `.helloagents/DESIGN.md` → `hello-ui`。

设计方向：
- 复刻静态页已经稳定下来的黑夜 / 白天视觉语言：黑夜为深色霓虹绿色科技感，白天为基于 logo 主色调的柔和、温暖、清醒亮色方案。
- 不修改静态页布局语义，但 React 首页可按组件职责拆分实现；最终视觉结构应与静态页一致。
- Hero 插图继续作为首屏记忆点，黑夜与白天都要有适配的底色扩散渐变，不出现边界硬切或阴影残留。
- 卡片阴影在白天模式保持轻、低饱和，黑夜模式保持玻璃感但不脏。
- 公告、API 接入地址、模型状态、价格、FAQ 等区块需要真实信息层级，不能为撑页面增加伪内容。
- 动效遵循已确认方向：滚动入场按从上到下、从左到右的顺序；主题切换为全屏连续过渡，不做三段式卡顿。
- 所有交互控件必须有可见焦点、aria label 或可读文本。

需要更新 `.helloagents/DESIGN.md` 的稳定规则：
- 正式首页与静态 SEO 页共享同一套 aiapi114 品牌首页视觉原则：黑夜霓虹绿色科技感、白天柔和温暖清醒。
- 首页主题切换动画和 Hero 插图是品牌记忆点，但必须支持减弱动效。

## 风险与验证
主要风险：
- 静态页 CSS 体量大，直接迁移会造成维护困难和样式污染。
- 首页同时接入公告、模型状态、主题和 i18n，容易出现加载顺序和兜底状态遗漏。
- 多语言文件体量较大，批量编辑可能引入 JSON 格式错误。
- 视觉复刻容易与现有 PublicLayout / 全局主题变量冲突。

验证方式：
- 纯逻辑测试：公告归一化、模型状态摘要映射、主题状态工具。
- 项目验证：`npm run typecheck`、`npm run test:model-status`、必要时新增首页测试命令、`npm run build`。
- 浏览器验收：`http://127.0.0.1:3173/` 或实际 dev server 的 `/`，覆盖桌面/移动、黑夜/白天、中文/英文、接口失败兜底。
- 静态页回归：访问 `/prototypes/home-html/`，确认仍能展示。

回退点：
- 保留旧 `Home` 入口逻辑，若新版默认首页出现问题，可回退 `src/features/home` 的改造提交。
- 静态 SEO 页不参与改造回退，不作为 React 首页运行依赖。
