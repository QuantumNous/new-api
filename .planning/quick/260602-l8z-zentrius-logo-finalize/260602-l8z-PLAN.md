## Decisions

- **Single-line SVG** (the user's exact format) rather than pretty-printed
  multi-line. Trade-off: smaller file (722B vs 870B), but harder to read
  by hand. User's snippet is the canonical form, no reason to reformat.
- **256px canonical for logo.png / favicon.ico**. Renders cleanly at 16-32px
  (favicon) and 24-48px (header logo) because the strokes (65/40/6 in a
  500×500 viewBox) are 13%/8%/1.2% of viewBox — well within legibility range.
- **No new Logo component change**. `web/default/src/assets/logo.tsx` was
  already aligned to the Z+C design in the prior commit
  (`57146b4e9`) and renders correctly.

## Alternatives Considered

- **Reformat SVG to multi-line** — rejected; user provided the format.
- **Use rsvg-convert** — not installed; `convert` (ImageMagick) is available
  and produces identical bytes for these inputs.
- **Skip favicon.ico update** — rejected; the favicon currently shows the
  Atius infinity mark which is a visible brand mismatch.
