## 核心数据结构
### RelayInfo (relay/common/relay_info.go)
```
type RelayInfo struct {
    // 用户信息
    UserId, TokenId int
    UserGroup, UsingGroup string
    
    // 模型信息
    OriginModelName string    // 原始模型名
    UpstreamModelName string  // 上游模型名(映射后)
    
    // 渠道信息
    ChannelMeta *ChannelMeta  // 渠道元数据
    
    // 请求信息
    RelayMode int             // 转发模式
    RelayFormat types.RelayFormat // 请求格式
    IsStream bool             // 是否流式
    
    // 计费信息
    Billing BillingSettler    // 计费会话
    PriceData types.PriceData // 价格数据
}
```
## 关键流程说明
### 1. 渠道选择逻辑
- 根据 用户分组 和 模型名称 查找可用渠道
- 支持 渠道亲和性 (Channel Affinity)保持会话一致性
- 支持 多Key轮询 和 自动禁用 故障渠道
### 2. 请求格式转换
项目支持多种API格式的相互转换：

- OpenAI格式 → 各厂商格式 (主流)
- Claude格式 → OpenAI格式
- Gemini格式 → OpenAI格式
### 3. 计费流程
```
预扣费 → 请求转发 → 实际消费计算 → 结算/退款
```
### 4. 重试机制
- 最多重试 common.RetryTimes 次
- 根据HTTP状态码判断是否重试
- 支持跨分组重试
## 文件结构总结
目录 职责 router/ HTTP路由定义 middleware/ 认证、分发、限流中间件 controller/ 请求处理入口 relay/ 转发核心逻辑 relay/channel/ 各厂商适配器实现 relay/common/ 公共数据结构 service/ 业务逻辑(渠道选择、计费等) model/ 数据模型(GORM)

这个架构的核心优势是 适配器模式 ，使得添加新的AI服务商只需实现 Adaptor 接口即可，无需修改核心转发逻辑。
