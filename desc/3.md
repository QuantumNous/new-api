# 安全审计功能分析报告
## 📋 功能概述
本项目的安全审计功能可以记录所有API请求的完整内容（包括请求体、文件等），并保存到日志文件中，便于安全审查和追溯。
## 🔍 核心实现位置
### 1️⃣ 中间件层 - 请求拦截与数据收集
文件位置： middleware/audit.go这是安全审计的核心入口，作为Gin中间件在请求处理流程中拦截数据：
go
`````
func AuditMiddleware() gin.HandlerFunc {
    auditLogger := audit.GetAuditLogger()
    
    return func(c *gin.Context) {
        // 1. 检查是否启用审计
        if !auditLogger.IsEnabled() {
            c.Next()
            return
        }
        
        // 2. 记录开始时间
        startTime := time.Now()
        requestID := c.GetString(common.RequestIdKey)
        
        // 3. 提取请求体数据
        var requestBody []byte
        var files []audit.AuditFile
        
        // 处理 multipart/form-data（文件上传）
        if strings.HasPrefix(contentType, "multipart/form-data") {
            requestBody, files = extractMultipartData(c)
        } else if c.Request.Body != nil {
            // 读取普通JSON请求体
            bodyBytes, err := io.ReadAll(c.Request.Body)
            if err == nil {
                requestBody = bodyBytes
                c.Request.Body = io.NopCloser(bytes.NewBuffer(bodyBytes))
                
                // 从请求体中提取嵌入的文件（如base64图片）
                files = extractEmbeddedFiles(bodyBytes)
            }
        }
        
        // 4. 包装ResponseWriter以捕获响应
        blw := &bodyLogWriter{body: bytes.NewBufferString(""), ResponseWriter: c.Writer}
        c.Writer = blw
        
        // 5. 继续处理请求
        c.Next()
        
        // 6. 如果响应状态码>=400，不记录
        if c.Writer.Status() >= 400 {
            return
        }
        
        // 7. 构建审计记录
        record := &audit.AuditRecord{
            RequestID:   requestID,
            Timestamp:   startTime,
            TokenKey:    maskTokenKey(...),  // 脱敏的token密钥
            TokenID:     ...,
            UserID:      ...,
            Model:       c.GetString("original_model"),
            RelayMode:   c.GetInt("relay_mode"),
            RequestBody: json.RawMessage(requestBody),
            Files:       files,
            Metadata: map[string]interface{}{
                "client_ip":      c.ClientIP(),
                "user_agent":     c.GetHeader("User-Agent"),
                "request_method": c.Request.Method,
                "status_code":    c.Writer.Status(),
                "latency_ms":     time.Since(startTime).Milliseconds(),
                "channel_id":     ...,
                "channel_type":   ...,
                "channel_name":   ...,
            },
        }
        
        // 8. 异步记录到存储
        gopool.Go(func() {
            auditLogger.Log(record)
        })
    }
}
`````
#### 关键特性：
* ✅ 支持多种Content-Type（JSON、multipart/form-data）
* ✅ 提取嵌入的base64文件（如图片）
* ✅ Token密钥脱敏处理（显示为 sk-xxxx_xxxx）
* ✅ 只记录成功请求（状态码<400）
* ✅ 异步写入，不影响主请求性能

### 2️⃣ 服务层 - 审计日志管理
文件位置： service/audit/service.go

这个文件实现了审计日志的存储逻辑，支持两种模式：
#### A. 本地存储模式 (Local Storage)
go
`````
type LocalStorage struct {
    basePath    string      // 基础路径: logs/audit
    maxFileSize int64       // 单文件最大大小
    currentFile string      // 当前文件路径
    currentSize int64       // 当前文件大小
}

func (ls *LocalStorage) Save(record *AuditRecord) error {
    // 1. 按Token创建目录（脱敏后的token作为目录名）
    tokenDir := filepath.Join(ls.basePath, maskTokenKey(record.TokenKey))
    os.MkdirAll(tokenDir, 0755)
    
    // 2. 按日期创建子目录
    dateDir := filepath.Join(tokenDir, record.Timestamp.Format("2006-01-02"))
    os.MkdirAll(dateDir, 0755)
    
    // 3. 检查是否需要创建新文件（超过maxFileSize时轮换）
    if ls.currentFile == "" || ls.currentSize >= ls.maxFileSize {
        ls.currentFile = ls.getNewFilePath(dateDir)
        ls.currentSize = 0
    }
    
    // 4. 将记录序列化为JSON并追加写入
    data, _ := json.Marshal(record)
    data = append(data, '\n')  // JSONL格式
    
    f, _ := os.OpenFile(ls.currentFile, os.O_APPEND|os.O_CREATE|os.O_WRONLY, 0644)
    f.Write(data)
    
    // 5. 如果有文件附件，单独保存
    if len(record.Files) > 0 {
        filesDir := filepath.Join(dateDir, "files", record.RequestID)
        os.MkdirAll(filesDir, 0755)
        
        for i, file := range record.Files {
            filename := fmt.Sprintf("%d_%s", i, file.Filename)
            fp := filepath.Join(filesDir, filename)
            decoded, _ := base64.StdEncoding.DecodeString(file.Base64Data)
            os.WriteFile(fp, decoded, 0644)
        }
    }
}
`````

#### B. 远程服务模式 (Remote Storage)
go
`````
type RemoteStorage struct {
    endpoint string  // 远程服务地址
    apiKey   string  // API密钥
    timeout  time.Duration
}

func (rs *RemoteStorage) Save(record *AuditRecord) error {
    // 使用multipart/form-data发送审计记录
    writer := multipart.NewWriter(&buf)
    
    // 添加JSON记录
    jsonData, _ := json.Marshal(record)
    writer.WriteField("record", string(jsonData))
    
    // 添加文件附件
    for i, file := range record.Files {
        decoded, _ := base64.StdEncoding.DecodeString(file.Base64Data)
        part, _ := writer.CreateFormFile(fmt.Sprintf("file_%d", i), file.Filename)
        part.Write(decoded)
    }
    
    // 发送HTTP POST请求
    req, _ := http.NewRequest("POST", rs.endpoint+"/api/audit/log", &buf)
    req.Header.Set("Authorization", "Bearer "+rs.apiKey)
    
    resp, _ := rs.client.Do(req)
}
`````

#### 🔄 完整工作流程

plaintext
`````
用户请求
   ↓
[Gin Router] → middleware.AuditMiddleware()
   ↓
1. 检查审计是否启用
   ↓
2. 读取请求体（JSON或multipart）
   ↓
3. 提取嵌入的文件（base64图片等）
   ↓
4. 包装ResponseWriter捕获响应
   ↓
5. c.Next() → 继续处理请求（更换token、转发给上游）
   ↓
6. 请求完成后，检查状态码
   ↓
7. 构建AuditRecord对象
   ↓
8. gopool.Go() → 异步写入存储
   ↓
   ├─→ LocalStorage: 写入 logs/audit/{token}/{date}/audit_xxx.jsonl
   └─→ RemoteStorage: POST到远程服务
`````

💡 关键设计亮点

* 性能优化：使用 gopool.Go() 异步写入，不阻塞主请求
* 文件轮换：单个日志文件达到 MaxFileSize 后自动创建新文件
* 结构化存储：按Token和日期分层组织，便于检索
* 敏感信息保护：Token密钥脱敏显示
* 多格式支持：同时支持JSON和multipart/form-data
* 文件提取：自动从请求体中提取base64编码的文件
* 错误隔离：审计失败不影响主业务流程
* 自动清理：根据保留天数自动删除旧日志


## 📌 总结

安全审计功能通过 Gin中间件 在请求处理的早期阶段拦截数据，在请求完成后异步记录到存储系统。本地存储模式下，日志按 {token}/{date}/ 的层级结构保存在 logs/audit 目录中，采用JSONL格式便于后续分析和检索。

### ⚠️ 注意事项
代码作为普通文本消息时，不会被提取为独立文件，但会在 request_body 中完整记录
建议使用 Base64 或 Multipart 方式上传文件，这样可以保留文件名和类型信息
大文件受 MaxFileSize 限制（默认 10MB），超过会被跳过
修改后，所有常见的程序员源代码文件都能被审计日志正确识别、保存和恢复了！🎉
