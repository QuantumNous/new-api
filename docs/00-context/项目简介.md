# 项目简介

## 一句话定位

eLight 不是单纯卖 Token，而是为企业提供全球主流 AI 模型的**统一接入、交付与运营服务**，让客户更稳定、更省心地把 AI 用到真实业务里。

## 我们看到的问题

今天很多企业并不是不想用全球最强的 AI 模型，而是很难真正稳定地用起来：

- 模型分散，能力各不相同。
- 采购链路复杂，支付与结算成本高。
- 账号、额度、权限、风控都不稳定。
- 缺少统一服务窗口，出了问题没人负责。
- 很多企业拿到了模型，却没有真正落地到业务流程中。

企业真正缺的不是一个账号，而是一套可以持续使用的 AI 接入与服务体系。

## 两大业务场景

### 场景一：企业自有统一 AI 接入平台

面向有明确 IT 治理需求的企业，提供私有化或半托管的 AI API 网关：

- 统一企业所有 AI 接入，聚合 40+ 上游提供商。
- 与企业 SSO 集成，按员工身份鉴权。
- 按部门/组织结构管理用量和账单。
- 企业自主管理渠道、模型、Token、配额和权限。

### 场景二：eLight 在线 AI Token 服务

面向不需要（或暂时无力）自建统一接入平台的企业和团队：

- 由 eLight 运营统一接入基础设施，客户直接消费。
- 提供 API Credits、订阅代充、团队账户等交付形式。
- 覆盖全球主流模型，提供高速稳定的 Token 服务。
- 通过多模型、多渠道冗余降低中断风险。

两个场景共享同一套技术平台，区别在于部署模式和运营主体。

## 四层核心能力

### 1. 统一接入

- 覆盖 OpenAI、Claude、Gemini、Azure、AWS Bedrock 等 40+ 上游平台。
- 帮助客户按场景选择合适模型。
- 降低企业多平台对接成本。

### 2. 统一交付

- 提供 API Credits、订阅代充、团队账户等交付能力。
- 客户不必分别处理多个上游平台的采购和结算。
- 提升接入效率与交付确定性。

### 3. 统一运营

- 统一处理额度补给、问题响应、使用支持。
- 通过多模型与多渠道能力降低中断风险。
- 帮客户把"能用"变成"持续稳定地用"。

### 4. 统一服务

- 提供使用辅导、Prompt 工程、场景落地、持续运营。
- 不只帮客户接入模型，也帮助客户把 AI 放进具体业务流程里。
- 帮客户从试用阶段走向生产阶段。

## 为什么有价值

客户向 eLight 购买的，不只是额度本身，而是：

- 更低的接入复杂度（多模型采购与整合）。
- 更高的可持续性（统一交付与统一结算）。
- 更明确的服务责任（问题响应与持续服务）。
- 更快的业务落地（面向企业场景的落地支持）。
