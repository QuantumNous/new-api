# 核心业务流程

完整产品边界见 [组织与账单产品设计](./组织与账单产品设计.md)。

## 个人账号与扣费流程

系统真实扣费主体始终是个人用户账号。

```text
请求 -> Token -> User -> 个人钱包/订阅 -> 预扣 -> 结算/退款 -> Log
```

组织扩展不进入这条链路，不新增组织预扣、组织结算、组织退款或组织消费流水。

## 创建组织

系统管理员在 `/admin/organizations` 创建组织：

1. 输入组织名称。
2. 后端创建 `Organization`，状态默认为启用。
3. `owner_id = 0`，不创建任何 `OrganizationMember`。

创建组织不会让任何用户加入组织，也不会修改用户账号、余额、订阅、Token 或 `group`。

## 添加组织成员

系统管理员或组织 Owner/Admin 可以把已有用户加入组织：

1. 在组织成员页点击 `Add member`。
2. 搜索并选择一个已存在的启用用户。
3. 选择角色。
4. 后端校验该用户当前不属于其他组织。
5. 后端创建 `OrganizationMember`，记录 `joined_at`，并保持 `left_at = 0`。

角色规则：

- 系统管理员可以添加 `owner`、`admin`、`billing` 或 `member`。
- 组织尚无 Owner 时，系统管理员可以把某个成员设为 Owner，并回填 `Organization.owner_id`。
- 组织 Owner/Admin 只能添加 `admin`、`billing` 或 `member`，不能指定 Owner。

## 移除组织成员

移除成员只结束组织成员关系：

1. Owner/Admin 触发二次确认。
2. 后端禁止移除 Owner。
3. 后端禁止当前操作者移除自己，避免组织无人管理。
4. 后端设置 `left_at`，不物理删除成员关系。

移除成员不删除用户账号，不修改 Token，不影响个人余额、订阅、充值、退款或历史日志。

## 组织账单查询

组织账单只解释已经发生的个人消费日志：

```text
Organization -> OrganizationMember.UserId -> Log.UserId
```

统计范围按成员有效期过滤：

- `Log.CreatedAt >= OrganizationMember.JoinedAt`
- `OrganizationMember.LeftAt = 0` 或 `Log.CreatedAt < OrganizationMember.LeftAt`

成员加入组织前的个人消费不进入组织账单；成员离开后，组织仍可查看其成员有效期内产生的历史消费。
