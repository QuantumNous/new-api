# 用户与角色

## 账号角色

- 普通用户：拥有个人账号、个人余额、个人订阅、个人 Token 和个人消费日志。
- 系统管理员：管理用户、渠道、模型、兑换码、订阅、系统配置和全局组织。
- Root：拥有完整系统配置权限。

## 组织角色

组织角色独立于账号角色，不改变用户本身的系统角色。

- Owner：由系统管理员在组织成员管理中显式指定，管理组织基础信息、成员和角色，可查看组织账单。
- Admin：管理本组织成员和角色，可查看组织账单。
- Billing：可查看和导出组织账单，不能管理成员。
- Member：默认只能查看自己在组织成员有效期内的用量。

## 组织边界

组织是“成员集合 + 报表权限 + 汇总视图”，不是付款主体。

- 组织不持有余额、订阅、充值订单或 API Key。
- 用户加入组织不改变个人余额、个人订阅、Token、扣费、退款和日志写入链路。
- `group` 是用户分组、模型可用性、路由和倍率维度，不是组织。
- 用户请求时不能选择组织，Token 和 Header 也不表达组织归属。
- 未加入组织的用户没有当前组织，不显示组织入口。

## 加入组织规则

用户只在组织成员关系被显式创建时加入组织：

- 管理员创建组织时只创建组织记录，不添加任何用户。
- 系统管理员在组织详情中添加成员时，目标用户加入该组织；组织尚无 Owner 时，系统管理员可以把该成员设为 Owner。
- 组织 Owner/Admin 在本组织成员页添加成员时，目标用户加入该组织。

注册、OAuth 首次登录、管理员创建系统用户、登录、创建 Token、充值和订阅都不会自动加入组织。

一个用户同一时间只能属于一个组织。添加成员时必须校验目标用户存在、启用、未删除，且当前没有 `left_at = 0` 的组织成员关系。
