# 验收标准

## 组织管理

- 管理员能只输入组织名称创建空组织。
- 创建组织后不会自动添加任何用户，`owner_id = 0`。
- 管理员能将未加入组织的启用用户加入指定组织。
- 组织尚无 Owner 时，管理员能将一个未加入组织的启用用户设为 Owner。
- 组织 Owner/Admin 能在用户侧成员页添加成员、修改角色和移除非 Owner 成员。
- Billing 能查看和导出组织账单，但看不到成员管理操作。
- Member 默认只能看到自己在组织成员有效期内的用量。
- 未加入组织的用户不显示组织入口，直接访问组织页面时显示未加入组织的空状态。

## 无侵入边界

- 管理员创建组织和创建系统用户时都不能隐式加入组织；如需加入组织，必须通过组织成员接口完成。
- 注册、登录、创建 Token、充值、订阅和 API 请求不会自动加入组织。
- 组织功能不修改用户 `group`、个人余额、个人订阅、Token 归属、扣费、退款或日志写入链路。
- UI 不出现组织余额、组织充值、组织订阅、组织 API Key、Header 组织归属或项目成本中心。

## 成员关系与账单

- 一个用户同一时间只能属于一个组织。
- 添加成员时必须拒绝已属于其他组织的用户。
- 移除成员只设置 `left_at`，保留历史成员关系。
- 组织账单按成员有效期汇总现有个人消费日志。
- 成员加入组织前的历史消费不进入组织账单。
- 成员离开组织后，组织仍能查看其成员有效期内产生的历史消费。
- 组织明细导出复用当前筛选条件。
