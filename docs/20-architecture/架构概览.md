# 架构概览

## 分层结构
系统遵循 Router -> Controller -> Service -> Model 的分层结构。

## 主要模块
- `router/`：HTTP 路由入口，包含 API、relay、dashboard 与 web 路由。
- `controller/`：请求处理与接口响应。
- `service/`：业务逻辑与跨模块编排。
- `model/`：GORM 数据模型与数据库访问。
- `relay/`：AI API 代理与 provider 适配。
- `setting/`、`common/`、`dto/`、`types/`：配置、公共工具、数据结构与类型定义。
- `web/default/`：默认前端。
