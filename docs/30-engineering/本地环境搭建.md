# 本地环境搭建

## 一键启动

仓库根目录的 `start.sh` 同时启动 Go 后端和 `web/default` 前端，并把日志写入 `logs/dev/backend.log` 与 `logs/dev/frontend.log`：

```bash
./start.sh
```

默认端口：

- 后端：`3000`
- 前端：`3001`

自定义端口：

```bash
BACKEND_PORT=3002 FRONTEND_PORT=3003 ./start.sh
```

脚本会检查 Go、Bun，创建缺失的前端嵌入占位资源，并在缺少依赖时从 `web/` 安装默认前端依赖。退出脚本时会结束本轮启动的后端、前端和日志跟随进程。

## 可选本地依赖

需要 PostgreSQL 与 Redis 时：

```bash
START_DEPS=1 ./start.sh
```

脚本使用 Docker 启动或复用命名容器，并默认设置：

- PostgreSQL 15，宿主机端口 `5432`
- Redis 7，宿主机端口 `6379`
- `SQL_DSN` 与 `REDIS_CONN_STRING`

通过 `POSTGRES_*`、`REDIS_*` 环境变量覆盖容器、端口和凭据。脚本退出后依赖容器会继续运行。

## 手动启动

后端：

```bash
go run . --port 3000 --log-dir ./logs/backend
```

前端：

```bash
cd web/default
VITE_REACT_APP_SERVER_URL=http://localhost:3000 bun run dev -- --host 0.0.0.0 --port 3001
```

前端包管理统一优先使用 Bun。
