# 产品定位

## 目标受众

### 主要客户

- **有 IT 治理需求的企业**：需要统一管理内部 AI 接入，按部门/员工审计用量和费用。
- **中小企业与技术团队**：不需要自建平台，但需要稳定、多模型的 AI Token 服务。

### 次要客户

- **独立开发者**：需要便捷获取多模型 API 访问。
- **AI 应用集成商**：需要可靠的上游 Token 供应和统一结算。

## 价值主张

eLight 把复杂、分散的 AI 模型使用流程，重构为一个标准化的企业服务入口。

客户获得的核心价值：

| 价值维度 | 自行对接上游 | 使用 eLight |
|---------|------------|-----------|
| 接入复杂度 | 每个提供商单独对接 | 一个 API、一个 Key |
| 采购与结算 | 多平台分别付款 | 统一结算 |
| 稳定性 | 单点依赖 | 多渠道冗余 + 自动 fallback |
| 服务责任 | 各平台各自负责 | 统一服务窗口 |
| 业务落地 | 自行摸索 | 提供场景咨询和落地支持 |

## 差异化

eLight 不做简单转售。差异化体现在：

1. **多模型整合能力**：40+ 提供商统一接入，客户无需逐一对接。
2. **企业级治理**：SSO 集成、组织管理、用量审计、权限隔离。
3. **服务而非资源**：从卖 Token 升级为卖接入稳定性和落地服务。
4. **灵活部署**：支持 SaaS 和私有化部署，适配不同企业安全要求。

## 不做什么

- 不做模型训练和微调。
- 不做模型推理基础设施（GPU 集群）。
- 不做通用 SaaS AI 产品（如 ChatGPT 竞品）。
- 不做面向 C 端消费者的服务。
