# 交互模式

## 页面结构

- 管理页使用页头、紧凑工具栏、单层 Panel 和表格，不堆叠多层卡片。
- 重要指标同时显示格式化 Amount 与可审计的 Raw Quota。
- 窄屏优先纵向重排，宽表格使用横向滚动，不静默裁剪字段。

## 表单与操作

- 创建、添加和设置使用 Dialog；危险删除使用 ConfirmDialog。
- 图标按钮必须提供 `sr-only` 文本或 `aria-label`。
- 提交中禁用重复操作，成功后关闭弹窗、刷新相关查询并给出 toast。
- 服务端拒绝原因应原样映射为可理解错误，不只显示无反馈的 disabled 状态。

## 数据状态

- Loading、empty、error、disabled、permission denied 分开呈现。
- 筛选变化时重置分页；刷新和导出复用同一组筛选。
- 权限必须由后端保证，菜单隐藏和页面拒绝只作为前端体验层。

## 可访问性

- 使用语义化标题、表格、按钮和表单标签。
- 状态不能只用颜色表达。
- 保留键盘焦点、合理 Tab 顺序、WCAG AA 对比度和 reduced motion。
