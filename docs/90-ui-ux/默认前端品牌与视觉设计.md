# 默认前端品牌与视觉设计

## 定位

`web/default` 的本地视觉核心是 Lighting/eLight 企业 AI 接入服务，而不是通用 API 网关模板。功能协议、权限、状态和可访问性继续跟随上游；品牌、默认首页信息架构和视觉 token 以本地设计为真源。

根路径默认标题、meta title 和 favicon 使用 Lighting；控制台、登录页与其他默认品牌入口使用 Lighting 名称和紧凑裁剪的 `lighting-mark.png`。后端未配置品牌，或仍返回旧默认 `New API` / `/logo.png` / `/lighting-favicon.png` / `/lighting-logo.png` 时，前端视为未自定义并迁移到 Lighting；其他显式配置的名称和 Logo 仍保留。favicon 会在 HTML 首屏、本地缓存恢复和后台状态刷新三个阶段保持一致。多个 `copy*.html` 是设计过程参考，不是运行时真源。

## 默认首页

没有配置自定义首页内容时，默认首页区块顺序固定为：

```text
Hero -> Why -> Scenarios -> Capabilities -> Governance
     -> Workflow -> Ecosystem -> CTA -> HomeFooter
```

首页表达企业模型统一接入、渠道冗余、计费结算、组织报表与持续服务。区块文案均走 i18n，链接通过 `HomeLink` 保留站内/站外行为。

### 必须保留的可配置分支

本地首页只替换默认内容，不能覆盖上游自定义首页能力：

- URL：沙箱 iframe，并同步语言和已解析主题。
- HTML：隔离的 `RichContent` HTML 渲染。
- Markdown：公共布局中的 `RichContent` Markdown 渲染。

同步或重构 `features/home/index.tsx` 时，先保留这三种分支，再调整默认 Lighting 首页。

## 视觉系统

首页使用 `--home-*` 与 `home-*` 命名空间：

- 主色：品牌蓝。
- 辅色：青色。
- 文本：深海军蓝与柔和次级文本。
- 背景：浅色渐变、细网格和低对比边界。
- 表面：适度圆角、柔和阴影、清晰层级。
- 动效：短时、克制，并尊重 reduced motion。

公共头部使用 reference variant 和滚动后的紧凑胶囊呈现。移动端仍需保留菜单、鉴权、通知、语言和动态导航行为。

## 登录入口

登录、注册、忘记密码、OTP、重置和 OAuth 回调共享 `auth-page` / `auth-card`：

- 与首页一致的品牌蓝、青色渐变和细网格背景。
- 居中的 440px 上限品牌卡片。
- Logo 与系统名置于卡片顶部。
- 主提交按钮使用品牌蓝和适度高光。

视觉统一不能改变各认证页面的字段、校验、OAuth/OTP 流程、错误提示和可访问性。

## 浅色主题决策

当前工作区把 ThemeProvider 固定为 light，清理旧 `html.dark` 并保留 `useTheme()` 兼容接口；主题切换入口已从公共头部、命令菜单和相关页面移除。

该决策只表示产品不提供 dark/system 切换，不自动授权删除其他个性化能力。配置抽屉原来还承载字体、preset、圆角、缩放、侧边栏样式、折叠方式、内容宽度和方向；这些能力应逐项决定保留、迁移或移除。

上游新增主题或可访问性状态时，必须先判断它是视觉偏好还是功能依赖，不能因为 light-only 决策整块丢弃。

## 同步真源

本地应保留：

- Lighting/eLight Logo、favicon、根路径默认标题。
- 默认首页区块顺序和企业服务叙事。
- `home-*`、`auth-*` token 与视觉合同。
- 公共头部 reference variant 和登录品牌卡片。
- 已确认的 light-only 产品决策。

上游应优先保留：

- 自定义首页 URL/HTML/Markdown 分支。
- 鉴权、通知、移动菜单、动态导航和登录倒计时。
- API、权限、路由、缓存、表单校验和错误处理。
- 键盘操作、focus、disabled、loading、reduced motion 和响应式功能状态。

混合文件不得整文件选边，详细步骤见 [上游同步与冲突解决指南](../30-engineering/上游同步与冲突解决指南.md)。

## 设计资产治理

- 运行时使用的图片放在 `web/default/public/`。
- 最终设计说明和可复用参考放在 `docs/90-ui-ux/`。
- 临时 HTML 过程稿不作为实现验收依据；确定最终稿后应归档或清理多余副本。
- locale 文件按 key 合并，不能用整个文件覆盖远端新功能翻译。
- `routeTree.gen.ts` 属于生成文件，先解决源路由再由构建重新生成。
