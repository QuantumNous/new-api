Codex 图像生成 API（gpt-image-2）

通过标准 OpenAI 图像接口（/v1/images/generations 与 /v1/images/edits）调用 gpt-image-2 生成或编辑图片。请求/响应与 OpenAI Images API 兼容，可直接迁移；结果以 b64_json 形式返回，并按上游真实用量（image token）按量计费。

1. 概览

项值
可用模型gpt-image-2（以你的服务实际开放的模型名为准）
调用模式同步：一次请求直接返回图片（无需轮询）
能力文生图（generations）、图生图 / 局部编辑（edits，支持可选 mask）
请求格式与官方 OpenAI Images API 一致：generations 为 JSON，edits 为 multipart/form-data
结果交付图片以 base64 返回（data[].b64_json），附 revised_prompt（模型改写后的提示词）
计费按 token（上游真实 image-gen 用量），随 quality / size 自动分档

下文中的 {BASE} 指服务地址，例如 https://your-api-host。

2. 认证

所有请求需在请求头携带访问令牌：

Authorization: Bearer sk-xxxxxxxxxxxxxxxx

3. 接口一览

方法 & 路径说明Content-Type鉴权
POST/v1/images/generations文生图application/jsonBearer
POST/v1/images/edits图生图 / 编辑（含可选 mask）multipart/form-dataBearer

要让某个 codex 渠道提供图像能力，需在该渠道的「模型」列表中加入 gpt-image-2。承载该图像请求的内部文本模型默认 gpt-5.4，可在渠道设置 / 全局设置中覆盖，对客户端透明。

4. 文生图

POST{BASE}/v1/images/generations

请求参数（JSON Body）

字段类型必填说明
modelstring必填固定 gpt-image-2
promptstring必填图像描述
sizestring可选输出尺寸，如 1024x1024
qualitystring可选画质档位：low / medium / high（不传由上游默认）。档位越高 token / 费用越多，见第 6 节
backgroundstring可选背景处理，如 transparent / opaque（透传上游）
output_formatstring可选图片格式，如 png / jpeg / webp（透传上游）
output_compressionint可选压缩级别（透传上游）
moderationstring可选内容审核档位（透传上游）
response_formatstring可选仅支持 b64_json（默认）。传 url 会被拒绝，见第 7 节
nint可选不生效：每次请求固定返回 1 张，见第 7 节

请求示例

curl {BASE}/v1/images/generations \
  -H "Authorization: Bearer sk-xxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-image-2",
    "prompt": "a photorealistic red fox in a snowy forest at golden hour, highly detailed",
    "size": "1024x1024",
    "quality": "high"
  }'

响应

{
  "created": 1781096207,
  "data": [
    {
      "b64_json": "iVBORw0KGgoAAAANSUhEUgAA...（base64 编码的 PNG）...RK5CYII=",
      "revised_prompt": "A photorealistic red fox sitting in a snowy forest at golden hour...",
      "url": ""
    }
  ]
}

图片在 data[0].b64_json（base64）。url 字段恒为空字符串——本接口只回 base64，不提供托管 URL。revised_prompt 为模型实际使用的改写提示词。

5. 图生图（编辑）

POST{BASE}/v1/images/edits · multipart/form-data

表单字段

字段类型必填说明
modeltext必填固定 gpt-image-2
prompttext必填编辑指令
imagefile必填待编辑图片（PNG / JPEG / WEBP）。多图可用 image[] / image[0]…
maskfile可选蒙版图：透明区域为待重绘区域。提供后做局部重绘（inpainting）
sizetext可选输出尺寸
qualitytext可选画质档位，同第 4 节

若提供了 mask 但服务端无法读取它（损坏 / 超限），请求会直接报错，不会退化成整图编辑——避免「编辑结果错误却照常计费」。

请求示例

curl {BASE}/v1/images/edits \
  -H "Authorization: Bearer sk-xxxx" \
  -F model=gpt-image-2 \
  -F prompt="add a small yellow star in the center" \
  -F image=@input.png \
  -F mask=@mask.png

响应

{
  "created": 1781096428,
  "data": [
    {
      "b64_json": "iVBORw0KGgoAAAANSUhEUgAA...（编辑后的 PNG, base64）...",
      "revised_prompt": "Edit the provided image: add a small yellow five-pointed star exactly in the center...",
      "url": ""
    }
  ]
}

edits 的输入图本身也会计入 token 用量（image_gen.input_tokens 含输入图 image-token），因此 edits 的输入 token 通常高于纯文生图。

6. 质量与计费

计费按 token：上游返回的 image-gen 用量（input + output image-token），随 quality / size 自动分档。下表为 1024×1024 的实测量级（具体单价由你的服务倍率配置决定）：

quality输出 image-token（约）相对成本典型耗时
standard / 默认~1961×（基准）~17–19s
high~1700+≈ 9×~60–70s
edits（含输入图）输入 ~256 + 输出按档输入图额外计入~45s

定价注意：一张 high 质量图的成本约为默认质量的 9 倍。若客户端可自由指定 quality:high，请据此设定合理倍率，避免成本失控。

部分订阅档位上游可能将 high 实际降级为中等画质——这是上游账号档位行为，网关如实透传 quality 并按上游返回的真实 token 计费。

7. 限制与约定

约定说明
每请求 1 张上游图像工具不接受 n 参数，故 n>1 仍只返回 1 张（不报错）。需要多张请多次调用
仅 b64_json不提供托管 URL；response_format:"url" 会被拒绝（HTTP 400）
模型前缀图像接口要求 gpt-image-* 模型；其他模型名会被拒绝
白标上游错误不透传给客户端：失败时仅返回通用错误消息，上游品牌 / host / 内部模型名不外泄
高质量延迟high 出图可达 ~60–70s，请确保客户端 / 代理 / 网关超时足够（建议 ≥150s）

8. 错误

请求 / 提交阶段错误走标准错误体：

{ "error": { "message": "...", "type": "...", "code": "..." } }
典型场景说明
response_format:"url"HTTP 400：codex image: response_format "url" not supported; codex image only supports b64_json
非 gpt-image-* 模型图像接口要求 gpt-image-* 模型
edits 缺少 imagemultipart 中未提供图片文件
上游失败（4xx/5xx）返回通用错误消息（白标），详情记录在服务端日志
额度不足请求被拒

9. 完整示例（文生图 → 落地为图片文件）

#!/usr/bin/env bash
BASE="https://your-api-host"
TOKEN="sk-xxxxxxxxxxxxxxxx"

# 文生图，取出 b64 并解码为 PNG
curl -sS "$BASE/v1/images/generations" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-image-2",
    "prompt": "a cute corgi puppy sitting on grass",
    "size": "1024x1024"
  }' \
| jq -r '.data[0].b64_json' \
| base64 -d > output.png

echo "saved -> output.png"
