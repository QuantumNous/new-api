异步视频生成 API

Seedance 视频生成 API

调用视频生成接口生成视频(文生 / 图生)。采用「提交任务 → 轮询状态 → 取回视频」的异步模型;视频经本服务代理返回,可直接嵌入页面。

1. 概览

项值
可用模型seedance-2.0、seedance-2.0-fast(以你的服务实际开放的模型名为准)
调用模式异步:创建任务 → 轮询 → 取视频
能力文生视频、图生视频(传入参考图)
结果交付视频经本服务代理地址返回,可直接下载或嵌入

下文中的 {BASE} 指服务地址,例如 https://your-api-host。

2. 认证

除「获取视频」接口外,所有请求需在请求头携带访问令牌:

Authorization: Bearer sk-xxxxxxxxxxxxxxxx

例外:「获取视频」接口 GET {BASE}/v1/videos/{task_id}/content 为匿名接口,无需令牌 —— 不可猜的 task_id 本身即访问凭证,可直接用于 <video src> 嵌入。

3. 接口一览

方法 & 路径说明鉴权
POST/v1/video/generations创建视频生成任务Bearer
GET/v1/video/generations/{task_id}查询任务状态/结果Bearer
GET/v1/videos/{task_id}/content取回视频二进制(代理下载)匿名

4. 创建视频任务

POST{BASE}/v1/video/generations

请求参数(JSON Body)

字段类型必填说明
modelstring必填模型名,如 seedance-2.0(以你的服务开放的名称为准)
promptstring必填视频提示词
durationint可选时长(秒),整数,如 5
resolutionstring可选分辨率,如 "1080p"
ratiostring可选画面比例,如 "16:9"
imagesstring[]可选图生视频的参考图(取第一张),URL 或 base64

注意:仅识别上述参数。watermark、seed、generateAudio、returnLastFrame 等参数不会生效(即使传入也会被忽略)。

请求示例(文生视频)

curl {BASE}/v1/video/generations \
  -H "Authorization: Bearer sk-xxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "seedance-2.0",
    "prompt": "宇航员在月球漫步，电影感，镜头缓慢推近",
    "duration": 5,
    "resolution": "1080p",
    "ratio": "16:9"
  }'

请求示例(图生视频)

curl {BASE}/v1/video/generations \
  -H "Authorization: Bearer sk-xxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "seedance-2.0",
    "prompt": "让画面里的人物缓缓转头微笑",
    "duration": 5,
    "images": ["https://example.com/ref.jpg"],
    "resolution": "1080p",
    "ratio": "16:9"
  }'

响应

{
  "id": "task_3f9a...e2",
  "task_id": "task_3f9a...e2",
  "object": "video",
  "model": "seedance-2.0",
  "status": "queued",
  "progress": 0,
  "created_at": 1780306574
}

id / task_id 是公开任务 ID(task_ 前缀 + 32 位随机串),后续轮询和取视频都用它。

5. 查询任务状态

GET{BASE}/v1/video/generations/{task_id}

curl {BASE}/v1/video/generations/task_3f9a...e2 \
  -H "Authorization: Bearer sk-xxxx"

排队 / 进行中

{
  "id": "task_3f9a...e2",
  "object": "video",
  "status": "in_progress",
  "progress": 50,
  "model": "seedance-2.0",
  "created_at": 1780306574
}

完成

{
  "id": "task_3f9a...e2",
  "object": "video",
  "status": "completed",
  "progress": 100,
  "model": "seedance-2.0",
  "metadata": {
    "url": "{BASE}/v1/videos/task_3f9a...e2/content"
  },
  "created_at": 1780306574,
  "completed_at": 1780306750
}

完成后,视频地址在 metadata.url —— 它是本服务的代理地址。直接 GET 该地址即可拿到视频(见第 6 节)。

失败

{
  "id": "task_3f9a...e2",
  "object": "video",
  "status": "failed",
  "error": { "message": "..." }
}

失败时不计费(已预扣的额度会退回)。

6. 获取视频

GET{BASE}/v1/videos/{task_id}/content · 匿名

任务成功后,直接 GET 该地址即可流式返回 video/mp4 二进制。无需鉴权,可直接嵌入页面:

# 下载到本地
curl -L "{BASE}/v1/videos/task_3f9a...e2/content" -o output.mp4

# 或在 HTML 中直接嵌入
<video src="{BASE}/v1/videos/task_3f9a...e2/content" controls></video>

仅当任务状态为 completed 时可取;未完成会返回错误。该接口按客户端 IP 限流。

7. 状态值

status含义
queued排队中
in_progress生成中(progress 0–100)
completed已完成,视频在 metadata.url
failed失败,详情在 error.message;已退款

8. 计费

按次计费:

  • 提交任务时按模型价格全额预扣额度。
  • 任务 completed → 保留扣费。
  • 任务 failed / 超时 / 查不到 → 退还预扣额度。

9. 错误

任务级失败体现在查询响应的 error 字段(见第 5 节)。请求/提交阶段的错误走标准错误体:

{ "error": { "message": "...", "type": "...", "code": "..." } }
典型场景说明
缺少 prompt提交被拒,prompt 必填
额度不足预扣失败,任务不会创建
上游暂时不可用code: bad_response_status_code 等,通常可稍后重试

10. 完整示例(创建 → 轮询 → 下载)

#!/usr/bin/env bash
BASE="https://your-api-host"
TOKEN="sk-xxxxxxxxxxxxxxxx"

# 1) 创建任务
RESP=$(curl -sS "$BASE/v1/video/generations" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "seedance-2.0",
    "prompt": "宇航员在月球漫步，电影感",
    "duration": 5,
    "resolution": "1080p",
    "ratio": "16:9"
  }')
TASK_ID=$(echo "$RESP" | jq -r '.id')
echo "task: $TASK_ID"

# 2) 轮询直到完成或失败
while true; do
  Q=$(curl -sS "$BASE/v1/video/generations/$TASK_ID" \
        -H "Authorization: Bearer $TOKEN")
  STATUS=$(echo "$Q" | jq -r '.status')
  echo "status: $STATUS"
  [ "$STATUS" = "completed" ] && break
  if [ "$STATUS" = "failed" ]; then
    echo "failed: $(echo "$Q" | jq -r '.error.message')"; exit 1
  fi
  sleep 5
done

# 3) 取回视频(匿名,直接用 metadata.url)
URL=$(echo "$Q" | jq -r '.metadata.url')
curl -L "$URL" -o output.mp4
echo "saved -> output.mp4"
