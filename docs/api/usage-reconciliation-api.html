用量对账 API

BlockRun 渠道用量对账 API

按时间窗拉取 BlockRun 系渠道下各模型 / 各 API Key 的 token 消耗与折后成本，用于对账。提供两个只读接口：/usage/summary（聚合汇总）与 /usage/transactions（逐条明细，分页）。

1. 概览

项值
用途BlockRun 系渠道（渠道类型名前缀 blockrun，大小写不敏感）消耗对账
接口GET /usage/summary（聚合）、GET /usage/transactions（明细，分页）
口径仅统计成功计费日志（消耗类）；token 含输入 / 输出 / 缓存读 / 缓存写；成本为折后实收
币种恒为 USD；金额为定点 10 位小数的字符串
时间入参 start / end 为 RFC3339（UTC）；区间 end > start 且 ≤ 31 天
幂等纯只读 GET，可安全重试

下文中的 {BASE} 指服务地址，例如 https://router.flatkey.ai。

2. 认证

两个接口都需在请求头携带静态共享令牌（与上游账户无关，仅鉴权、不区分用户）：

Authorization: Bearer <token>

只接受 Authorization: Bearer 形式（不支持 ?token= 查询参数或自定义请求头）。Bearer 方案名大小写不敏感。令牌由服务方通过安全渠道下发。

令牌缺失或错误 → 401；服务端未配置令牌 → 503。请妥善保管令牌，勿写入前端代码 / 公开仓库。

3. 接口一览

方法 & 路径说明鉴权
GET/usage/summary时间窗内的聚合用量（总计 + 按模型 + 按 API Key）Bearer
GET/usage/transactions时间窗内的逐条用量明细（分页）Bearer

4. 公共参数（时间窗）

两个接口共用以下查询参数：

参数类型必填说明
startstring必填起始时间，RFC3339（UTC），如 2026-06-01T00:00:00Z。含该时刻
endstring必填结束时间，RFC3339（UTC）。不含该时刻；须 end > start

区间长度 end - start 须 ≤ 31 天，否则返回 400。需要更长周期请分段拉取。

5. 用量汇总

GET{BASE}/usage/summary

返回该时间窗内 BlockRun 系渠道的聚合用量：总计 totals、按模型 by_model、按 API Key by_api_key。三处都使用同一套指标对象（见下）。

请求示例

curl -G "{BASE}/usage/summary" \
  -H "Authorization: Bearer <token>" \
  --data-urlencode "start=2026-06-01T00:00:00Z" \
  --data-urlencode "end=2026-06-08T00:00:00Z"

响应字段

字段类型说明
providerstring固定 "flatkey-newapi"
periodobject{ start, end, timezone }，回显查询窗口，timezone 恒为 "UTC"
totalsobject（指标）全窗口总计，见「指标对象」
by_api_keyarray按 API Key 拆分；每项 = api_key_id + api_key_name + 指标对象；按 requests 降序
by_modelarray按模型拆分；每项 = model + 指标对象；按 requests 降序
generated_atstring响应生成时刻，RFC3339（UTC）

指标对象（totals / by_model[] / by_api_key[] 共用）

字段类型说明
requestsint请求数（计费日志条数）
input_tokensint输入 token
output_tokensint输出 token
cache_read_tokensint缓存读 token
cache_creation_tokensint缓存写 token（含 5m/1h 合计）
total_tokensint= input + output + cache_read + cache_creation
actual_coststring折后实收成本（USD），定点 10 位小数。详见第 7 节
currencystring恒为 "USD"

6. 用量明细（分页）

GET{BASE}/usage/transactions

返回时间窗内的逐条用量明细，按 created_at, id 升序，分页返回。

额外参数

参数类型必填说明
pageint可选页码，从 1 起；默认 1。非法值回落为默认
page_sizeint可选每页条数，默认 100，上限 500（超出按 500 处理）

外加第 4 节的 start / end。

请求示例

curl -G "{BASE}/usage/transactions" \
  -H "Authorization: Bearer <token>" \
  --data-urlencode "start=2026-06-08T00:00:00Z" \
  --data-urlencode "end=2026-06-08T01:00:00Z" \
  --data-urlencode "page=1" \
  --data-urlencode "page_size=100"

响应字段

字段类型说明
transactionsarray本页明细，见「明细元素」
paginationobject{ page, page_size, total_pages, total_count, has_more }
generated_atstring响应生成时刻，RFC3339（UTC）

明细元素 transactions[]

字段类型说明
transaction_idstring明细 ID，"txn_" + 日志 id
request_idstring请求 ID
api_key_idstringAPI Key ID
api_key_namestringAPI Key 名称
modelstring实际上游模型名（缺省时同 requested_model）
requested_modelstring客户端请求的模型名
created_atstring发生时刻，毫秒 UTC（...000Z）
input_tokensint输入 token
output_tokensint输出 token
cache_read_tokensint缓存读 token
cache_creation_tokensint缓存写 token
total_tokensint= input + output + cache_read + cache_creation
actual_coststring折后实收成本（USD），定点 10 位小数
currencystring恒为 "USD"
statusstring"success" 或 "error"
duration_msint处理耗时（毫秒，秒级精度）
metadataobject{ channel_id, channel_name }（渠道维度）

分页对象 pagination

字段类型说明
pageint当前页码
page_sizeint每页条数（已应用 500 上限）
total_pagesint= ceil(total_count / page_size)
total_countint命中总条数
has_morebool是否还有下一页

7. 字段语义与计费口径

主题说明
actual_cost折后实收成本（USD），= 内部额度 Quota / 500000，定点 10 位小数字符串。按字符串解析为十进制再求和，避免浮点误差
无 total_cost响应不包含 total_cost（折前官价）字段 —— 该口径此处无法取得，刻意不返回、也不以实收值伪充
缓存 tokencache_read_tokens = 缓存命中读取；cache_creation_tokens = 写入缓存（含 5m / 1h 合计）
model vs requested_modelrequested_model 为客户端请求名；model 为实际上游模型名，二者可能不同（如别名 / 路由）
statussuccess 为默认；流式响应标记为错误时为 error
duration_ms处理耗时，毫秒；源为秒级，故为 1000 的整数倍
created_at明细时间为毫秒 UTC（...000Z）；period / generated_at 为秒级 RFC3339
metadata仅含 channel_id / channel_name（渠道维度），不暴露上游主机、内部链路等信息
数据范围仅统计 BlockRun 系渠道的消耗类计费日志；汇总不做渠道维度拆分（渠道信息见明细的 metadata）
排序by_model / by_api_key 按 requests 降序；transactions 按 created_at, id 升序

8. 错误与限流

错误响应体为简单 JSON：

{ "error": "..." }
状态码场景
400start / end 缺失或非 RFC3339；end ≤ start；区间 > 31 天
401缺少 / 错误的 Authorization: Bearer 令牌
429触发限流（按客户端 IP）。降低频率后重试
503服务端未配置令牌（运维未就绪）

page / page_size 的非法值不会报错：会回落为默认（page=1、page_size=100），page_size 超过 500 按 500 处理。

9. 完整示例（汇总 + 遍历明细）

#!/usr/bin/env bash
set -euo pipefail
BASE="https://router.flatkey.ai"
TOKEN="<token>"
START="2026-06-01T00:00:00Z"
END="2026-06-08T00:00:00Z"

# 1) 汇总
curl -sS -G "$BASE/usage/summary" \
  -H "Authorization: Bearer $TOKEN" \
  --data-urlencode "start=$START" \
  --data-urlencode "end=$END" | jq .

# 2) 遍历全部明细（按 has_more 翻页）
page=1
while :; do
  resp=$(curl -sS -G "$BASE/usage/transactions" \
    -H "Authorization: Bearer $TOKEN" \
    --data-urlencode "start=$START" \
    --data-urlencode "end=$END" \
    --data-urlencode "page=$page" \
    --data-urlencode "page_size=500")
  echo "$resp" | jq -c '.transactions[]'
  more=$(echo "$resp" | jq -r '.pagination.has_more')
  [ "$more" = "true" ] || break
  page=$((page + 1))
done
