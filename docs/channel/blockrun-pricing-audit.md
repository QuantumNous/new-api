# BlockRun 渠道定价验证报告

**测试时间：** 2026-05-20  
**测试目的：** 验证 BlockRun 渠道是否兑现"官价七折"承诺  
**测试者：** Claude Code（自动化链上余额对比）

---

## 一、背景与测试设计

BlockRun 是基于 x402 微支付协议的 AI API 渠道，使用以太坊钱包（Base 主网 USDC）按调用付费。  
渠道声称：**对接入方提供 Anthropic 官价七折（70%）优惠**。

new-api 网关配置：
- BlockRun 渠道接入，向上游付费
- 对用户分组设置**七五折（75%）**折扣

### 验证方法（方案一：余额差值法）

1. 记录调用前钱包 USDC 余额（Base 主网链上查询）
2. 通过 new-api 网关发送测试请求，获取实际 token 用量
3. 调用后多次轮询余额（每 6–7 秒一次，共 35 秒）捕获链上结算
4. 计算实际扣款 vs 官价 × 70% 的差异

### 官方定价参考

| 模型 | Input（官价） | Output（官价） | BlockRun 标价 |
|------|-------------|--------------|--------------|
| Claude Sonnet 4.6 | $3.00 / MTok | $15.00 / MTok | $3.00 / MTok（+5% markup）|
| Claude Opus 4.7 | $5.00 / MTok | $25.00 / MTok | $5.00 / MTok（+5% markup）|
| Claude Opus 4.5 | $5.00 / MTok | $25.00 / MTok | $5.00 / MTok（+5% markup）|

> BlockRun 页面注明："5% markup on official Anthropic API prices"，但标价与 Anthropic 官价一致，markup 机制存疑。

## 四、关键异常：Opus 定价不随 Token 数变化

| 对比 | Opus 4.7 小 | Opus 4.5 小 |
|------|-----------|-----------|
| Input Tokens | 685 | 466（相差 219） |
| max_tokens | 50 | 50 |
| **实际扣款** | **$0.006001** | **$0.006001** |

| 对比 | Opus 4.7 大 | Opus 4.5 大 |
|------|-----------|-----------|
| Input Tokens | 2,649 | 1,797（相差 852）|
| max_tokens | 100 | 100 |
| **实际扣款** | **$0.023022** | **$0.023022** |

**结论：BlockRun 对 Opus 系列按 `max_tokens` 分档计费，与实际 input token 数量、模型版本均无关。**  
这是一种"预授权最大费用"的计费模式，而非真实 per-token 计费，对小 context 大 max_tokens 的场景极度不利。

## 五、三方对账汇总

| 数据来源 | 金额 | 说明 |
|---------|------|------|
| **链上钱包实际扣款** | **$0.131771** | BlockRun 真实从钱包收走 |
| Anthropic 官价合计 | $0.079359 | 按官方定价 × token 用量 |
| **官价 × 70%（声称价格）** | **$0.055551** | BlockRun 承诺的价格 |
| 官价 × 75%（new-api 用户侧） | $0.059519 | 若新API按七五折向用户计费 |
| **new-api 渠道账本增量** | **$0.035800** | new-api 后台记录的渠道消耗 |

### 差异分析

```
BlockRun 实收 ÷ 官价        = 166.0%  （超出官价 66%，而非打七折）
BlockRun 实收 ÷ 官价×70%   = 237.2%  （超收声称价格的 2.37 倍）
钱包实际消耗 − new-api 记录 = $0.0960  （72.8% 的真实成本未进 new-api 账本）
```

## 六、结论

### 6.1 BlockRun 未兑现七折承诺

所有正式测试中，BlockRun 实际收费均**超过** Anthropic 官价，幅度在 128%–224% 之间，没有任何一次接近声称的 70% 水平。

### 6.2 new-api 成本核算严重失真

new-api 账本仅记录了 $0.0358 的渠道消耗，而链上实际扣款为 $0.1318，**72.8% 的真实成本（$0.0960）在 new-api 账本中不可见**。

这导致运营层面：
- new-api 以为 BlockRun 渠道的单次调用成本很低
- 实际链上的 USDC 支出远高于账面
- 以官价七五折向用户收费，却在以超过官价 66% 的价格采购，**每笔交易均在亏损**

### 6.3 定价模式与宣传不符

BlockRun 对 Opus 系列按 `max_tokens` 分档定价（而非按实际 token 数），用户无法通过压缩 input 来降低成本。

