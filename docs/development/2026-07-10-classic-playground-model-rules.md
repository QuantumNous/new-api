# 2026-07-10 Classic 操练场模型规则

## 变更范围

本次只为 `web/classic` 增加操练场模型规则能力，`web/default` 暂不支持。

## 入口位置

后台 `模型管理` 页面新增 `操练场模型规则` tab，用于配置 classic 操练场使用的模型排序和分类。

## 规则能力

每条规则按模型名精确匹配，支持以下字段：

- `categories`
  - 可选值：`chat`、`image`、`video`。
  - 支持多选，因此同一个模型可以同时出现在多个 classic 操练场分类中。
  - 一旦为某模型显式配置 `categories`，该模型的操练场分类将不再使用自动识别，而是完全以规则为准。
- `orders`
  - 为对象结构，按分类分别保存排序值，例如：
    - `orders.chat`
    - `orders.image`
    - `orders.video`
  - 每个分类的排序值互相独立，同一个模型在聊天、图片、视频三个分组里可以使用完全不同的排序数字。
  - 允许填写任意数字，例如 `10 / 20 / 30`，最终按数字升序排序。
  - 未配置某分类排序值时，该分类回退到模型名自然排序。

兼容性说明：

- 旧规则中的单个 `order` 字段继续兼容读取。
- 读取旧规则时，会把该 `order` 视为三个分类的默认排序值，避免已有配置失效。

## classic 前台生效逻辑

classic 操练场读取 `/api/status` 返回的 `playground_model_rules`，并缓存到浏览器本地。加载模型列表时：

1. 先应用后台配置的 `categories` 和 `orders`。
2. 未配置规则的模型，继续沿用原先的自动识别逻辑：
   - 聊天模型：基于 `supported_endpoint_types` 中的聊天端点类型判断；
   - 图片/视频模型：基于模型名关键字提示判断。
3. 三个分类列表分别独立排序：
   - 聊天列表使用 `orders.chat`
   - 图片列表使用 `orders.image`
   - 视频列表使用 `orders.video`
4. 每个分类最终先按该分类的自定义排序值升序，再回退到模型名自然排序。

## classic 后台交互

后台 `操练场模型规则` tab 分为两部分：

1. 分组设置
   - 控制模型是否出现在聊天、图片、视频三个 classic 操练场分组中。
   - 支持继续使用自动识别，也支持显式覆盖分类结果。
2. 独立排序
   - 按 `聊天模型`、`图片模型`、`视频模型` 三个分组分开展示。
   - 每个分组支持：
     - 直接拖拽调整顺序；
     - 手动填写序列号排序。
   - 拖拽和手填都会落到对应分类的排序值上，三个分组互不影响。

## 兼容性说明

- 本能力只影响 classic 操练场页面的模型展示与选择，不改变真实模型路由或后端转发行为。
- 后台配置保存后，classic 当前会话会同步更新本地缓存；其他用户在下一次刷新状态后生效。
